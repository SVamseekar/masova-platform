       179 +  ## 🎬 OBSERVED BEHAVIOR
       180 +  
       181 +  ### **When App Starts:**
       182 +  
       183 +  1. **User logs in** → Auth state populated with user info
       184 +  2. **Manager dashboard loads** → Uses `X-User-Store-Id` (their assigned store) initially
       185 +  3. **Manager selects different store** → Updates `selectedStoreId` in Redux
       186 +  4. **Next API call** → Sends `X-Selected-Store-Id` header with new store
       187 +  5. **Data refreshes** → Shows new store's data WITHOUT page reload ✨
       188 +  
       189 +  ### **Role-Based Filtering:**
       190 +  
       191 +  #### As MANAGER:
       192 +  - Can select any store from dropdown
       193 +  - Backend uses `X-Selected-Store-Id` header
       194 +  - Can switch between stores freely
       195 +  - Sees data for selected store
       196 +  
       197 +  #### As STAFF/DRIVER:
       198 +  - Store selector disabled (locked to assigned store)
       199 +  - Backend uses `X-User-Store-Id` header
       200 +  - Cannot access other stores' data
       201 +  - Always sees only their assigned store
       202 +  
       203 +  #### As CUSTOMER:
       204 +  - Selects store before ordering
       205 +  - Backend uses `X-Selected-Store-Id` header
       206 +  - Orders tied to selected store
       207 +
