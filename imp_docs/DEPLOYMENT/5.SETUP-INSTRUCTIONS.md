# 🚀 MaSoVa Update System - Setup Instructions

## ⚡ Quick Setup (30 Minutes)

Follow these steps in order to get your automated update system running.

---

## Step 1: Create Docker Hub Account (5 minutes)

1. Go to https://hub.docker.com/signup
2. Create a **FREE** account
3. Verify your email
4. **Write down your username** - you'll need it!

---

## Step 2: Configure MaSoVa (2 minutes)

Edit the `.masova-config` file:

```bash
# Open the config file
vim .masova-config

# Update these lines with YOUR information:
DOCKER_HUB_USERNAME=your-actual-dockerhub-username
GITHUB_USERNAME=your-actual-github-username
```

**Example:**
```bash
# If your Docker Hub username is "john123"
DOCKER_HUB_USERNAME=john123

# If your GitHub is github.com/john123/masova
GITHUB_USERNAME=john123
```

Save and close the file.

---

## Step 3: Test Local Build (5 minutes)

```bash
# Make build script executable
chmod +x build-monolith.sh

# Test build (without Docker)
./build-monolith.sh --skip-tests --version=1.0.0
```

**Expected output:**
```
✅ Parent POM built successfully
✅ user-service built and copied
✅ menu-service built and copied
... (all 13 services)
✅ Monolith build completed successfully!
```

**Check the output:**
```bash
ls -lh masova-monolith/
# Should see multiple .jar files
```

---

## Step 4: Setup GitHub Secrets (5 minutes)

### 4a. Create Docker Hub Access Token

1. Go to https://hub.docker.com/settings/security
2. Click "New Access Token"
3. Name: `GitHub Actions`
4. Permissions: Read, Write, Delete
5. **Copy the token** (you won't see it again!)

### 4b. Add Secrets to GitHub

1. Go to your GitHub repository
2. Click **Settings** → **Secrets and variables** → **Actions**
3. Click **New repository secret**

**Add Secret 1:**
- Name: `DOCKER_USERNAME`
- Value: Your Docker Hub username

**Add Secret 2:**
- Name: `DOCKER_PASSWORD`
- Value: The access token you just created

---

## Step 5: Create Your First Release (15 minutes)

```bash
# 1. Commit all changes
git add .
git commit -m "Setup automated update system"
git push

# 2. Create release tag
git tag v1.0.0
git push origin v1.0.0
```

### Watch GitHub Actions Build

1. Go to: https://github.com/YOUR_USERNAME/masova/actions
2. You should see "Build and Release MaSoVa" running
3. Wait 10-15 minutes (grab a coffee ☕)

### Verify Success

After GitHub Actions completes, check:

**✅ Docker Hub:**
- Go to: https://hub.docker.com/r/YOUR_USERNAME/masova-backend
- Should see tags: `1.0.0` and `latest`

**✅ GitHub Releases:**
- Go to: https://github.com/YOUR_USERNAME/masova/releases
- Should see release `v1.0.0`

## Step 6: Test Update Script (5 minutes)

```bash
# Create test directory
mkdir -p ~/MaSoVa-Test
cd ~/MaSoVa-Test

# Copy update script
cp ~/Projects/MaSoVa-restaurant-management-system/update-masova.sh .
chmod +x update-masova.sh

# Create simple docker-compose.yml for testing
cat > docker-compose.yml << EOF
version: '3.8'
services:
  backend:
    image: YOUR_USERNAME/masova-backend:latest
    ports:
      - "8080:8080"
    environment:
      - MONGODB_URI=mongodb://localhost:27017/masova
      - JWT_SECRET=test-secret
EOF

# Start service
docker-compose up -d

# Wait 30 seconds
sleep 30

# Test version endpoint
curl http://localhost:8080/api/system/version

# Should return: {"version":"1.0.0",...}
```

**Clean up test:**
```bash
docker-compose down
cd ..
rm -rf ~/MaSoVa-Test
```

## 🆘 Troubleshooting

### Build fails with "Maven not found"
```bash
# Install Maven
brew install maven  # macOS
```

### GitHub Actions fails
- Check secrets are added correctly in GitHub Settings
- Verify Docker Hub username matches exactly
- Check Actions logs for specific error

### Docker Hub push fails
```bash
# Test login
docker login
# Enter your Docker Hub username and password/token
```
