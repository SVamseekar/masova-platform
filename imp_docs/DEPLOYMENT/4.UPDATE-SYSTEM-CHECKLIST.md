## Phase 1: Prerequisites (5 minutes)

### 1.1 Development Tools

- [ ] Java 17+ installed
  ```bash
  java -version
  # Should show: openjdk version "17.x.x" or higher
  ```

- [ ] Maven installed
  ```bash
  mvn --version
  # Should show: Apache Maven 3.x.x
  ```

- [ ] Docker Desktop installed
  ```bash
  docker --version
  docker-compose --version
  # Both should work
  ```

- [ ] Git installed
  ```bash
  git --version
  ```

### 1.2 Accounts (FREE)

- [ ] GitHub account created
  - Sign up: https://github.com/signup
  - Verify email

- [ ] Docker Hub account created
  - Sign up: https://hub.docker.com/signup
  - Verify email
  - Create public repository: `masova`

### 1.3 Repository Setup

- [ ] GitHub repository created
  ```bash
  # If not already done:
  git init
  git add .
  git commit -m "Initial commit"
  git remote add origin https://github.com/YOUR_USERNAME/masova.git
  git push -u origin main
  ```

- [ ] Update scripts with your username
  - [ ] Edit `build-monolith.sh` line 10: `DOCKER_IMAGE="YOUR_USERNAME/backend"`
  - [ ] Edit `.github/workflows/release.yml` line 11: `DOCKER_IMAGE: YOUR_USERNAME/backend`
  - [ ] Edit `shared-models/.../UpdateService.java` line 25: Update GitHub URL

## Phase 2: Local Testing (10 minutes)

### 2.1 Test Build Script

- [ ] Make build script executable
  ```bash
  chmod +x build-monolith.sh
  ```

- [ ] Test build (without Docker)
  ```bash
  ./build-monolith.sh --skip-tests --version=1.0.0
  ```

- [ ] Verify output
  ```bash
  ls -lh masova-monolith/
  # Should see:
  # - Multiple .jar files
  # - start-masova.sh
  # - application.properties
  ```

### 2.2 Test Docker Build

- [ ] Build Docker image
  ```bash
  ./build-monolith.sh --docker --skip-tests --version=1.0.0
  ```

- [ ] Verify image
  ```bash
  docker images | grep masova
  # Should see:
  # masova/backend    1.0.0
  # masova/backend    latest
  ```

### 2.3 Test Docker Image Locally

- [ ] Create test docker-compose.yml
  ```bash
  cat > test-compose.yml << 'EOF'
  version: '3.8'
  services:
    mongodb:
      image: mongo:7
      ports:
        - "27017:27017"

    redis:
      image: redis:7-alpine
      ports:
        - "6379:6379"

    backend:
      image: masova/backend:1.0.0
      ports:
        - "8080:8080"
      environment:
        - MONGODB_URI=mongodb://mongodb:27017/masova
        - REDIS_HOST=redis
        - JWT_SECRET=test-secret-change-in-production
  EOF
  ```

- [ ] Start services
  ```bash
  docker-compose -f test-compose.yml up -d
  ```

- [ ] Wait 30 seconds and test health
  ```bash
  sleep 30
  curl http://localhost:8080/api/system/health
  # Should return: {"status":"UP", ...}
  ```

- [ ] Test version endpoint
  ```bash
  curl http://localhost:8080/api/system/version
  # Should return: {"version":"1.0.0", ...}
  ```

- [ ] Clean up
  ```bash
  docker-compose -f test-compose.yml down -v
  rm test-compose.yml
  ```

## Phase 3: Docker Hub Setup (5 minutes)

### 3.1 Docker Hub Login

- [ ] Login to Docker Hub
  ```bash
  docker login
  # Enter your Docker Hub username and password
  ```

- [ ] Create access token (recommended)
  - Go to: https://hub.docker.com/settings/security
  - Click "New Access Token"
  - Name: "GitHub Actions"
  - Permissions: Read, Write, Delete
  - Copy token (save it!)

### 3.2 Push Test Image

- [ ] Tag image
  ```bash
  docker tag masova/backend:1.0.0 YOUR_USERNAME/backend:1.0.0
  docker tag masova/backend:latest YOUR_USERNAME/backend:latest
  ```

- [ ] Push to Docker Hub
  ```bash
  docker push YOUR_USERNAME/backend:1.0.0
  docker push YOUR_USERNAME/backend:latest
  ```

- [ ] Verify on Docker Hub
  - Go to: https://hub.docker.com/r/YOUR_USERNAME/backend
  - Should see tags: `1.0.0` and `latest`

## Phase 4: GitHub Actions Setup (10 minutes)

### 4.1 Add Secrets to GitHub

- [ ] Go to your GitHub repository
- [ ] Click: Settings → Secrets and variables → Actions
- [ ] Click "New repository secret"

- [ ] Add `DOCKER_USERNAME`
  - Name: `DOCKER_USERNAME`
  - Secret: Your Docker Hub username

- [ ] Add `DOCKER_PASSWORD`
  - Name: `DOCKER_PASSWORD`
  - Secret: Your Docker Hub password or access token

### 4.2 Update Workflow File

- [ ] Edit `.github/workflows/release.yml`
  - Line 11: Change `masova/backend` to `YOUR_USERNAME/backend`
  - Line 12: Change `masova/frontend` to `YOUR_USERNAME/frontend`

- [ ] Commit changes
  ```bash
  git add .github/workflows/release.yml
  git commit -m "Configure GitHub Actions workflow"
  git push
  ```

### 4.3 Test GitHub Actions

- [ ] Create and push tag
  ```bash
  git tag v1.0.0
  git push origin v1.0.0
  ```

- [ ] Watch GitHub Actions
  - Go to: https://github.com/YOUR_USERNAME/masova/actions
  - Should see "Build and Release MaSoVa" running
  - Wait 10-15 minutes

- [ ] Verify completion
  - [ ] GitHub Actions: ✅ Green checkmark
  - [ ] Docker Hub: Images v1.0.0 available
  - [ ] GitHub Releases: v1.0.0 created
  - [ ] version.properties updated on main branch

## Phase 5: Update Scripts Testing (10 minutes)

### 5.1 Setup Test Installation

- [ ] Create test directory
  ```bash
  mkdir -p ~/MaSoVa-Test
  cd ~/MaSoVa-Test
  ```

- [ ] Create docker-compose.yml
  ```bash
  cat > docker-compose.yml << 'EOF'
  version: '3.8'

  services:
    mongodb:
      image: mongo:7
      container_name: masova-db
      volumes:
        - ./data/mongodb:/data/db
      ports:
        - "27017:27017"

    redis:
      image: redis:7-alpine
      container_name: masova-cache
      ports:
        - "6379:6379"

    backend:
      image: YOUR_USERNAME/backend:latest
      container_name: masova-backend
      ports:
        - "8080:8080"
      environment:
        - MONGODB_URI=mongodb://mongodb:27017/masova
        - REDIS_HOST=redis
        - JWT_SECRET=test-secret-change-this
      depends_on:
        - mongodb
        - redis

    frontend:
      image: YOUR_USERNAME/frontend:latest
      container_name: masova-frontend
      ports:
        - "3000:80"
  EOF
  ```

  - Replace `YOUR_USERNAME` with your Docker Hub username

- [ ] Copy update scripts
  ```bash
  cp ~/Projects/MaSoVa-restaurant-management-system/update-masova.sh .
  cp ~/Projects/MaSoVa-restaurant-management-system/rollback-masova.sh .
  chmod +x *.sh
  ```

- [ ] Create .env file
  ```bash
  cat > .env << 'EOF'
  JWT_SECRET=test-secret-for-testing-only
  FRONTEND_URL=http://localhost:3000
  MONGODB_URI=mongodb://mongodb:27017/masova
  REDIS_HOST=redis
  EOF
  ```

### 5.2 Test Initial Installation

- [ ] Start services
  ```bash
  docker-compose up -d
  ```

- [ ] Wait 30 seconds
  ```bash
  sleep 30
  ```

- [ ] Verify health
  ```bash
  curl http://localhost:8080/api/system/health
  curl http://localhost:8080/api/system/version
  ```

- [ ] Open in browser
  ```bash
  # Mac
  open http://localhost:3000

  # Linux
  xdg-open http://localhost:3000

  # Windows
  start http://localhost:3000
  ```

### 5.3 Test Update Process

- [ ] Release new version (v1.0.1)
  ```bash

  # Make a small change
  echo "# Version 1.0.1 test" >> README.md

  git push origin v1.0.1
  ```

- [ ] Verify update
  ```bash

  # Should show: "version":"1.0.1"
  ```

### 5.4 Test Rollback

- [ ] List backups
  ```bash

  ```

- [ ] Select v1.0.0


### 5.5 Cleanup Test

- [ ] Stop services
  ```bash
  docker-compose down -v
  ```

- [ ] Remove test directory
  ```bash

  ```

## Troubleshooting

### Build Fails

```bash
# Check Java
java -version

# Check Maven
mvn --version

# Clean and rebuild
mvn clean
./build-monolith.sh
```

### Docker Build Fails

```bash
# Check Docker
docker --version

# Clean Docker
docker system prune -a

# Rebuild
./build-monolith.sh --docker
```

### GitHub Actions Fails

- Check secrets are set correctly
- Verify workflow file syntax
- Check GitHub Actions logs
- Ensure Docker Hub credentials are valid

### Update Script Fails

- Check Docker Compose is installed
- Verify network connectivity
- Check Docker Hub for image availability
- Review script logs
