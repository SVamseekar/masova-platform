## ✅ Backend Documentation (Java Spring Boot)

### 1. Smart-doc Maven Plugin
**Location:** `pom.xml` (parent)

**What it does:**
- Automatically generates API documentation from Javadoc comments
- No special annotations required (unlike Swagger)
- Zero code invasion - uses standard Java comments
- Generates multiple formats: HTML, Markdown, OpenAPI 3.0, Postman Collection

**Configuration file:** `smart-doc.json`

**Generate docs:**
```bash
mvn smart-doc:html        # HTML documentation
mvn smart-doc:markdown    # Markdown documentation
mvn smart-doc:openapi     # OpenAPI 3.0 spec
mvn smart-doc:postman     # Postman collection
```

**Output location:** `target/smart-doc/`

## 💡 Best Practices

### For Backend Developers

**Write Javadoc comments:**
```java
/**
 * Creates a new order for a customer.
 *
 * @param orderRequest Order details including items and delivery info
 * @return Created order with ID and status
 * @throws OrderValidationException if order data is invalid
 */
@PostMapping("/orders")
public OrderResponse createOrder(@RequestBody OrderRequest orderRequest) {
    // implementation
}
```

**Use @Schema annotations:**
```java
public class OrderRequest {
    @Schema(description = "Customer ID", example = "cust_123")
    private String customerId;

    @Schema(description = "Store ID", example = "store_456")
    private String storeId;
}
```

### For Frontend Developers

**Write JSDoc comments:**
```typescript
/**
 * Menu item component displaying a single menu item
 * @param props - Component props
 * @param props.item - Menu item data
 * @param props.onAddToCart - Callback when item is added to cart
 */
export const MenuItem: React.FC<MenuItemProps> = ({ item, onAddToCart }) => {
    // implementation
};
```

## 🆘 Troubleshooting

### "Smart-doc: No such plugin"
```bash
# Clear Maven cache and reinstall
rm -rf ~/.m2/repository/com/ly/smart-doc
mvn clean install
```

### "TypeDoc command not found"
```bash
cd frontend
npm install typedoc --save-dev
```

### "Swagger UI shows 404"
```bash
# Make sure service is running
curl http://localhost:8081/actuator/health

# Check if springdoc dependency is present
grep -r "springdoc" */pom.xml
```

### "Documentation looks empty"
```bash
# Make sure you have Javadoc comments in your code
# Smart-doc needs comments to generate docs
```
